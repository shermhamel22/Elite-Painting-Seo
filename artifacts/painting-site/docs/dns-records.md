# DNS Records — elitepaintingsolutions.com

These records improve email deliverability (so your quote replies don't land in
spam) and lock down who is allowed to send mail "from" your domain. Add them in
your domain registrar's DNS panel (GoDaddy, Namecheap, Cloudflare, etc.).

> **Heads up:** if you only send mail through Gmail (the address
> `eps.paintingsolutions@gmail.com`), the SPF / DKIM / DMARC records below
> protect the **domain** `elitepaintingsolutions.com` from being spoofed.
> Gmail's `@gmail.com` address itself is already covered by Google's own DNS.
> Add these the day you start sending email from `you@elitepaintingsolutions.com`.

---

## 1. SPF — who is allowed to send email "from" your domain

Type:  `TXT`
Host:  `@`  (root of `elitepaintingsolutions.com`)
Value: `v=spf1 include:_spf.google.com ~all`
TTL:   `3600`

If you also send through another provider, add it to the same record. Examples:

- Google Workspace + Mailgun: `v=spf1 include:_spf.google.com include:mailgun.org ~all`
- Google Workspace + SendGrid: `v=spf1 include:_spf.google.com include:sendgrid.net ~all`

Only ONE SPF record per domain — merge into a single `v=spf1` string.

---

## 2. DKIM — cryptographic signature for outgoing mail

DKIM is generated by your email provider. For Google Workspace:

1. Open Google Admin → Apps → Google Workspace → Gmail → Authenticate email.
2. Click **Generate new record** (2048-bit).
3. Copy the host (e.g. `google._domainkey`) and the value (a long `v=DKIM1; ...`).
4. Add it as a `TXT` record in DNS.
5. Wait ~1 hour, then click **Start authentication** in Google Admin.

For other providers (Mailgun, SendGrid, Postmark) the steps are similar — the
provider gives you the exact `TXT` record to paste.

---

## 3. DMARC — what to do with mail that fails SPF / DKIM

Start in monitor-only mode for 2 weeks, review the reports, then tighten.

Type:  `TXT`
Host:  `_dmarc`
Value: `v=DMARC1; p=none; rua=mailto:eps.paintingsolutions@gmail.com; ruf=mailto:eps.paintingsolutions@gmail.com; fo=1; adkim=s; aspf=s; pct=100`
TTL:   `3600`

After 2 weeks of clean reports, change `p=none` → `p=quarantine`, and
eventually → `p=reject`:

```
v=DMARC1; p=reject; rua=mailto:eps.paintingsolutions@gmail.com; adkim=s; aspf=s; pct=100
```

---

## 4. MX — only if you're using Google Workspace email

Skip if email stays at `@gmail.com`. For Google Workspace on the domain:

| Priority | Value                  |
| -------- | ---------------------- |
| 1        | `smtp.google.com`      |

(Older Google MX setups used 5 hosts: `aspmx.l.google.com`, `alt1.aspmx…`, etc.
Either set works — `smtp.google.com` is the current recommended single record.)

---

## 5. CAA — only certain CAs can issue TLS certs for your domain

Type:  `CAA`
Host:  `@`
Value: `0 issue "letsencrypt.org"`
       `0 issue "pki.goog"`
       `0 iodef "mailto:eps.paintingsolutions@gmail.com"`
